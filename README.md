# Теория вероятностей и математическая статистика для начинающих IT-предпринимателей (учебное пособие)

Исходники учебного пособия по теории вероятностей и математической статистике в формате **Quarto Book** + сопровождающие материалы (примеры кода, данные, иллюстрации). Репозиторий предназначен для совместной работы через GitHub и редактирования в VS Code.

> Книга основана на многолетних наработках авторов; работа над ней началась более года назад и сейчас переносится из исходного формата в электронный Quarto. Быстрые изменения в главах связаны с переносом уже подготовленного материала.

Авторы: Н.Н. Зубов, П.С. Ткачев, С.З. Умаров.

## Быстрый старт

### Требования

- **Quarto** (CLI).
- **VSCode**
- Для сборки **PDF**: установленный LaTeX (TinyTeX/TeX Live). Рекомендуемый движок — **LuaLaTeX**. Для корректной кириллицы в PDF используются шрифты **Libertinus** (устанавливаются как TeX-пакет).

### Сборка и предпросмотр

Предпросмотр (live preview):

```bash
quarto preview
```

Сборка (HTML + PDF согласно `_quarto.yml`):

```bash
quarto render
```

Только HTML:

```bash
quarto render --to html
```

Только PDF:

```bash
quarto render --to pdf
```

Очистка артефактов сборки:

```bash
rm -rf _book .quarto
```

## Установка TeX-зависимостей (шрифты Libertinus)

Если при сборке PDF появляются ошибки вида `font … not found`, установите пакет шрифтов:

```bash
tlmgr install libertinus-fonts
```

(Команда `tlmgr` доступна в окружении TinyTeX/TeX Live. На macOS может потребоваться добавить TeX в `PATH`.)

## Структура репозитория

```txt
.
├── _quarto.yml
├── book
│   ├── chapters
│   └── images
├── code
│   ├── data
│   ├── lib
│   ├── scripts
│   └── snippets
├── index.qmd
├── LICENSE.md
├── README.md
├── references.bib
└── references.qmd

```

- `_quarto.yml` — конфигурация проекта Quarto Book (форматы вывода, оглавление, параметры PDF/HTML, шрифты и т.п.).
- `index.qmd` — входная страница книги (первый элемент `chapters`).
- `references.qmd` — страница списка литературы.
- `references.bib` — библиография в формате BibTeX.

### Книга (`book/`)

- `book/chapters/` — основной текст книги, главы сгруппированы по папкам.
  - В каждой папке главы — один файл `01-*.qmd` (раздел главы).
- `book/images/` — изображения, используемые в тексте книги (рисунки, схемы, экспортируемые диаграммы).

### Код и материалы (`code/`)

Единая папка для всего, что относится к воспроизводимым примерам и генерации артефактов.

- `code/snippets/` — **фрагменты, которые вставляются в книгу** (печатные примеры). Рекомендуемый формат — `.qmd`-вставки, подключаемые через include.
- `code/scripts/` — **служебные скрипты**: генерация рисунков/таблиц/датасетов. Результаты должны сохраняться в `book/images/` или `code/data/` (в зависимости от назначения).
- `code/lib/` — многократно используемые модули/утилиты (общие функции, которые вызываются из `scripts/` и/или `snippets/`).
- `code/data/` — небольшие входные/промежуточные данные для примеров и генераторов (только то, что разумно хранить в репозитории).

### Генерация рисунков

Рисунки генерируются отдельными скриптами по главам и сохраняются в `book/images/`.
Зависимости устанавливаются один раз из корня проекта:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Пример генерации:

```bash
python code/scripts/figures/ch01/generate_figures.py
```

### PDF-типографика и переносы

Настройки PDF вынесены в отдельные LaTeX-файлы:

- `book/tex/typography.tex` — типографика, переносы в формулах, абзацные отступы.
- `book/tex/hyphenation.tex` — пользовательские переносы слов.

### Инструкции и лицензия

- `LICENSE.md` — лицензия на материалы.
- `README.md` — описание проекта и инструкции для участников (данный файл).

## Рекомендованный процесс совместной работы

- Основная ветка: `main` (должна всегда собираться).
- Любые изменения — через Pull Request:
  - ветки: `feature/...`, `fix/...`, `docs/...`
  - один PR = одна логическая правка
  - лучше избегать массовых переформатирований без причины
  - все пути — относительные; результаты генерации сохранять только в `book/images/` или `code/data/`.
  - скрипты и примеры должны быть детерминированными (фиксировать seed, где применимо).
  - артефакты сборки (`_book/`, `.quarto/`) не коммитить!
